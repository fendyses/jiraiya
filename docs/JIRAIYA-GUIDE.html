Overview

What JIRAIYA Is

JIRAIYA is a memory-first orchestration layer on top of Claude Code. Instead of starting every session blank, it loads identity, relationship history, open reminders, and project state from a set of markdown files — then keeps a library of 40 feature modules on standby, each triggered by a phrase, an event, or a hook.

Architecture

Memory Model

Three layers cooperate: a fast-changing working layer, a durable identity layer, and a raw narrative log.

🧬 main/ — Identity Layer

  • main-memory.md — personality, relationship, bond
  • current-session.md — RAM, resets each session
  • reminders.md — cross-session follow-ups
  • decisions.md — append-only decision log
  • post-mortems.md — append-only failure log

📓 daily-diary/ — Narrative Log

  • current/YYYY-MM-DD.md — today's session writeups
  • archived/YYYY-MM/ — past months, auto-archived
  • Source material for Echo-Memory-Recall
  • Real timestamps only — never guessed

🗂️ memory/ + master-memory.md

  • master-memory.md — universal entry point, loader
  • memory/ — workspace-scoped auto-memory (this tool's own store)
  • Read first at session start, before anything else

Rules

Always-On Constraints

RuleWhy
Never modify memory files without user confirmationMemory is durable identity — accidental edits compound silently across sessions
Never run destructive git commands without explicit approvalForce-push / reset can erase work that isn't recoverable
Always check reminders.md at session startCross-session follow-ups are the main way commitments survive a reset

Audit

Known Gaps

Skill path mismatch. CLAUDE.md's "Installed Skills" table says skills live at plugins/ses-skills/skills/<name>/SKILL.md. That folder doesn't exist — every actual SKILL.md lives at Feature/<FolderName>/SKILL.md instead (one nested exception: Feature/Skill-Plugin-System/skills/skill-plugin-system/SKILL.md).
8 features have no SKILL.md yet. Auto-Load-Hook-System, Auto-Worker-System, Break-Reminder-System, Code-Sharp-System, Dream-Ideas-System, Meeting-System, Observation-System, Orchestration-System, Discipline-System, and Token-Guard-System are README-only — they rely on README-described behavior rather than a formal trigger file.
repo-switcher isn't in the CLAUDE.md skill table. The "/repo" trigger is documented separately in CLAUDE.md's own "Repository Switcher" section, but doesn't appear as a row in the "Installed Skills (Auto-Trigger)" table alongside the others.
post-mortems.md is currently empty. Noted in the file itself as "good thing" — no failures logged yet, so Post-Mortem-System has no historical data to draw on.

This guide reflects the repository state as read on 2026-06-16. Re-run the inventory after adding or moving SKILL.md files.
