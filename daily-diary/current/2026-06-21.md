## 2026-06-21 (Late Night - 00:52) - Dashboard Visual Effects & The Two-Loop Smoke Bug

### Session Summary
- Type: Feature / Bug Fix / Visual Polish
- Focus: Add a pack of ambient + battle visual effects to `agents/dashboard.html`, then chase down why the chimney smoke wouldn't render
- Outcome: Wind sway, grass/flowers, chimney smoke, water caustics, and battle FX all working; discovered a duplicate `setAnimationLoop` that had been silently killing an entire effects loop

### What Was Built
1. **Wind sway on trees** ‚Äî `patchWind`/`patchWindMat` inject a per-vertex sway into GLB materials via `onBeforeCompile`, driven by a shared `uTime` uniform; taller geometry sways more. Applied to village + forest trees.
2. **Waving grass + swaying flowers** ‚Äî canvas-drawn grass-blade tufts (70 quads) and upright flower blossoms (22), both wind-patched, pivots translated to the base so they sway from the ground.
3. **Chimney smoke** ‚Äî soft camera-facing puff planes rising/drifting/expanding/fading from each `chimney:true` house; emitters registered synchronously from building coords.
4. **Water caustics** ‚Äî horizontal shimmer (`offset.x` sway) + emissive pulse layered on the existing vertical flow for stream and pond.
5. **Battle effects pack** ‚Äî anime speed-lines on clash peaks + rhythmic micro-shake impact beats (started with white impact-frame flashes + chakra auras, both later removed at Fendy's request ‚Äî the flashing hurt his eyes, the chakra he just didn't like).

### Bug Fixes & Adjustments
- **Eye-hurting battle strobe** ‚Äî removed the full-screen white impact flash and the two per-clash full-screen tint fills; kept gentle shake only.
- **Red-roof house** ‚Äî was skewed by a 45¬∞ rotation; removed it and added `noSideWalls:true` to kill the edge-on "pole" + jutting side-panel artifacts.
- **`SkinnedMesh ... skinning set to false` console spam** ‚Äî two sources: main character materials needed `skinning=true`, and the gold-trail clones were skinned meshes casting shadows with plain materials ‚Üí set `castShadow=false` on the afterimages.
- **Water running too fast** ‚Äî halved all four flow speeds (stream 0.9‚Üí0.4, foam 1.6‚Üí0.7, pond 0.35‚Üí0.16, pond foam 0.7‚Üí0.32).

### The Big One ‚Äî Why The Smoke Wouldn't Render
A long debugging chase. Smoke emitters registered fine (console confirmed 3 emitters at correct coords) but nothing appeared. Tried, in order: bigger/brighter puffs, emit higher above roofs, `depthTest:false`, Sprite‚Üíplane-mesh (suspecting `logarithmicDepthBuffer` breaking sprites). Still nothing.

The breakthrough came from **stopping the guessing and adding diagnostics**: a static red marker plane (proved plane meshes DO render) + a puff-count log (which never fired). The puff-count log never firing meant `updateSmoke` was never being called ‚Äî even though it was in the animation loop. Root cause: **the dashboard has TWO `setAnimationLoop` calls, and the second (line ~2533) silently replaces the first (line ~1876)** where I'd placed all the effect updates. The windmill/water worked only because the second loop happened to duplicate that code. Fix: moved `windUniforms.uTime`, `caustics`, and `updateSmoke(dt)` into the live loop ‚Äî which also revived the tree sway and water caustics that had been silently dead too.

### Key Insights & Learning
- **When something registers correctly but never executes, suspect the execution path, not the data.** A second `setAnimationLoop` overriding the first is invisible until you check.
- **Diagnostics beat guesses.** The red-marker (does anything render?) + puff-counter (does the updater run?) split the problem in two and isolated it instantly ‚Äî should have reached for that several reloads earlier.
- `THREE.Sprite` can misbehave under `logarithmicDepthBuffer` in r128; camera-facing plane meshes are the safer billboard path.
- One fix can revive multiple features ‚Äî the loop fix brought smoke, wind sway, AND water caustics to life at once.

### Main Topics Discussed
1. **Signboard polish** ‚Äî "by Fendy SES" sign moved bottom-aligned/right, font tuned (43‚Üí21‚Üí32px).
2. **Removed agent-box click stats panel** ‚Äî clicking cards no longer opens battles/mood; cursor reverted to default.
3. **Live system-monitor gadgets** ‚Äî replaced the 5 agent cards with 4 boxes (CPU load + sparkline, RAM bar, Storage bar, Network up/down with live sparklines), styled to match a reference image.
4. **Data source journey** ‚Äî browsers can't read CPU/RAM, so first wrote a PHP endpoint; discovered (via README + aider history) the dashboard runs on VS Code Live Server / file://, which can't run PHP; switched to a standalone Python stats server with CORS.
5. **Pivot to ServBay** ‚Äî Fendy registered the site as `jiraiya.es`. Built a PHP login (`index.php` + `logout.php` + session-gated `agents/dashboard.php`), converted `dashboard.html` ‚Üí `dashboard.php`, and restored a same-origin PHP stats endpoint (no separate process needed). Credentials set to fendy / 199254.
6. **The php-fpm PATH bug** ‚Äî gadgets showed CPU 100%, RAM "0 KB", net 0 B/s. Root cause: ServBay php-fpm runs with a minimal PATH missing `/usr/sbin` and `/sbin`, so `sysctl`/`netstat`/`route` silently failed (storage worked because `df` is in `/bin`). Fixed with one `putenv('PATH=...')` line.
7. **Removed agent terminal panels** ‚Äî the "FIGHT MAX FORCE" fake terminals; `pushLine` already guarded on a missing element so the Phaser scene was untouched.
8. **File split refactor** ‚Äî extracted CSS, the system-monitor JS, and the Three.js village IIFE into `agents/css/dashboard.css`, `agents/js/monitors.js`, `agents/js/village3d.js`. `dashboard.php`: 2749 ‚Üí 1395 lines.

### Key Insights & Learning
- **Match the fix to how the app is actually served.** Half the work was discovering the runtime (Live Server vs ServBay) ‚Äî the PHP-vs-Python-vs-PHP zigzag came from learning the deployment story late. Check how something is hosted *before* picking the backend.
- **Symptom patterns are diagnostic.** "Storage right, everything else wrong" pointed straight at a PATH problem ‚Äî `df`/`ps`/`vm_stat` live in `/bin`+`/usr/bin`, while `sysctl`/`netstat`/`route` live in `/usr/sbin`+`/sbin`. php-fpm strips the latter.
- **Verify on the real server, not just the CLI.** The PHP endpoint passed `php -l` and `php -S` but failed under php-fpm; reproducing with a restricted PATH (`env -i PATH=/usr/bin:/bin`) nailed it, and a curl to live `jiraiya.es` confirmed the fix.
- **Verbatim extraction beats retyping.** Split 2700 lines via a single Python pass by line range ‚Äî zero transcription risk, identical behavior, verified by re-fetching each asset (200) and loading the authed page.
