## 2026-06-20 (Early Morning - 01:16) - 3D Three.js Village Background for the Dashboard

### Session Summary
- Type: Feature / Graphics
- Focus: Replace the flat Phaser tile background of `agents/dashboard.html` with a 3D low-poly fantasy village rendered behind the walking agent characters
- Outcome: Working 3D village scene, iterated over many rounds to match a painted reference; characters now collide with buildings/trees

### What Was Built
1. **Three.js 3D background layer** behind the Phaser character canvas:
   - Added `three@0.128.0` + `GLTFLoader` via plain CDN `<script>` tags (importmap/ES-module approach failed — switched to globals).
   - New `<canvas id="bg3d">` at `z-index:0`; Phaser canvas made transparent on top (`transparent:true` + `setBackgroundColor('rgba(0,0,0,0)')`).
   - Loads GLB models from `assets/kenney_fantasy-town-kit_2.0/Models/GLB format/`.
2. **Principled building assembler** — measured every GLB's real bounding box first (walls are thin `0.1×1×1` modular panels on a 1-unit grid). Scale 1.0 everywhere + integer-grid snapping + auto-grounding by bounding box. This fixed the earlier "broken/floating/sinking" buildings.
3. **Scene composition** matching a painted reference (left→right): gazebo, windmill, green-roof sign house ("All 5 agents operational" canvas-texture sign), red-roof house, green-roof mill house with water wheel + flowing stream.
4. **Animation**: windmill blades + water wheel face the camera (wrapped in pivot groups) and spin; stream uses a scrolling canvas water texture.
5. **Look/tone**: rounded forest-green hills (flattened spheres, not cones), dense pine forest backdrop, warm sandy clearing, ACES tone mapping to kill white blowout, warm-tinted grey rocks, glowing blue point-light lanterns.
6. **Collision**: characters can no longer walk onto houses/trees — invisible ellipse obstacle zones (normalized to W/H) over the 5 buildings + 2 foreground trees; `isBlocked()` check in `AgentNPC.update()` and `pickTarget()`.

### Key Insights & Learning
- **`file://` blocks GLB loading** (CORS) — the 3D scene only works served over HTTP. Added `agents/serve.sh` (serves from repo root so `../main/repos.js` + `diary-data.js` resolve; opens `localhost:8777`).
- **The black screen** was Phaser's `setBackgroundColor(0x00000000)` being read as opaque black, covering the 3D — fixed with `'rgba(0,0,0,0)'`.
- **White blowout** = overexposed lighting (ambient 1.6 + sun 2.8 clipping light surfaces). Fixed with ACES tone mapping + reduced light intensities.
- A low-poly kit can match a painted reference's *composition/palette/props* but never its painted brushwork — set that expectation with the user explicitly.
- Measuring GLB bounding boxes (parse the glTF JSON chunk's POSITION accessor min/max) beats guessing model dimensions blindly.

### Earlier in the day (continued from 06-19 work)
- Dashboard terminal button now auto-launches `claude .` (AppleScript `do script`), and the CLI icon uses the real `claude-logo.svg`.

### Looking Forward

## 2026-06-20 (Afternoon - 12:30) - 3D Mini-Characters Polish & Interaction Fixes

### Session Summary
- Type: Bug Fix / Polish
- Focus: Fix 3D mini-character placement, walking direction, drag-and-drop, Jiraiya diary click, golden trail, and diary date format in `agents/dashboard.html`
- Outcome: All five issues resolved; characters face camera correctly, drag works via Three.js raycasting, diary opens on Jiraiya click

### What Was Fixed

1. **Coordinate mapping (characters invisible / wrong position)**
   - Root cause: linear `toWorld(px, py)` approximation was wrong — perspective camera makes the mapping non-linear.
   - Fix: replaced with `phaserTo3D(px, py)` using `THREE.Raycaster` + ground plane (`y=0`) intersection. Each Phaser position now ray-casts through the exact NDC coordinate and intersects the ground, giving the precise world position that projects back to the same screen pixel.

2. **Characters walking backwards**
   - Root cause: `baseRY: Math.PI` rotated all models 180° to face away from camera.
   - Fix: changed to `baseRY: 0` (face camera). Walk-turn angles changed from `225°/135°` to `−45°/+45°`.

3. **Drag-and-drop broken**
   - Root cause: Phaser hit area `Ellipse(0, -40, 70, 100)` uses texture-local coords — `(0, -40)` was outside the 64×64 frame entirely.
   - Fix: replaced with DOM-level + Three.js raycasting on `#game-wrap`. `pointerdown` raycasts all 5 character meshes; `pointermove` (>6px threshold) drags `npc.sprite.setPosition()` directly so nametag follows; `pointerup` releases and resumes walking.

4. **Jiraiya diary click broken**
   - Root cause: previous `_didMove` flag triggered on any `pointermove` with button held — even a 1px tremor killed the click.
   - Fix: unified into `setupCharacterInteraction`. Uses `dx²+dy²>36` to distinguish click from drag. Pure click on Jiraiya's 3D model → `showDiaryBubble(npc)`.

5. **Jiraiya golden shadow shows old pixel-art character**
   - Root cause: `emitTrail()` cloned the old pixel-art sprite texture (invisible with alpha=0).
   - Fix: `emitTrail()` now calls `window._spawnGoldTrail()` which deep-clones Jiraiya's Three.js GLB scene, overrides materials with gold `MeshBasicMaterial`, fades out over 480ms in the render loop.

6. **Diary date format → DD-MM-YYYY**
   - Added `fmtDate(s)` helper; diary book left panel now shows `20-06-2026` instead of `2026-06-20`.

### Key Insight
Phaser's custom hit area uses texture-local coords (0,0 = frame top-left). The most reliable hitbox for a 3D model behind a transparent 2D canvas is to bypass Phaser input entirely and use Three.js `Raycaster.intersectObject` on DOM events that bubble up from the canvas to the parent wrapper.

## 2026-06-20 (Evening - 18:43) - 3D Dashboard Visual Polish: River, House, Shadows & Timer

### Session Summary
- Type: Visual Polish / Bug Fix
- Focus: Multiple visual improvements to the `agents/dashboard.html` Three.js 3D scene
- Outcome: Cleaner river, better house orientation, fixed shadows, removed Pomodoro timer

### What Was Done

#### 1. Red-Roof House Placement & Orientation
- Moved the red-roof house from `z = -8.5` → `z = -14` (set back further behind the other buildings)
- Added `ry` rotation parameter to `building()` function so any building can be rotated on Y axis
- Tried `ry: Math.PI/2` first (pure side view), then refined to `ry: Math.PI/4` (45°) to achieve the 3/4 isometric look Fendy wanted — matching a reference picture of a cozy red-roof cabin with door on the right and side wall visible simultaneously

#### 2. River / Water Upgrade
- Upgraded `makeWaterTex()`: 128×128 canvas, deep blue gradient (navy → cobalt), 9 wavy flow lines at varying opacity, 4 bright highlight strokes, 16 sparkle dots
- Added new `makeFoamTex()`: white foam streaks and blobs on transparent background
- Main stream: widened 3.5→4.8 units, switched to `MeshStandardMaterial` with `roughness:0.05`, `metalness:0.25`, dark blue emissive glow
- Added foam overlay plane scrolling 1.6× faster with `AdditiveBlending`
- Pond: enlarged radius 2.2→3.0, same rich material
- Added pond foam cap, 3 concentric ripple ring geometries
- Added blue `PointLight(0x3DAEFF)` above the pond for water glow
- Added 18 sparkle particle circles along the stream banks

#### 3. Shadow System Overhaul (multi-round)

**Round 1 — Remove double-shadow:**
- Problem: every object was getting TWO shadows — a directional light shadow (angled) AND a fake-AO circle (centered). User chose to keep the real directional shadows, remove the circles.
- Removed the entire FAKE AMBIENT OCCLUSION block (all `addAO()` calls)
- Restored `castShadow = true` on all objects

**Round 2 — Clean building shadows:**
- Problem: each building's 8–16 individual wall panels cast messy overlapping thin shadow strips
- Solution: added invisible box shadow-caster (`MeshBasicMaterial({ visible: false })`) inside each building group — one clean shadow per building
- Set `bGroup.userData.noShadowCast = true` and wired `place()` to check this flag, so individual wall/roof panels receive shadows but don't cast them
- Added windows to side walls and back walls on upper storeys (was previously front-face only) — fixes red-roof house 3/4 view showing bare wood on the visible side face

**Round 3 — Remove watermill & right-side tree shadows:**
- Added `noShadow` param to `placeSpinner()`, applied to watermill wheel
- Added `noShadow: true` to right-side trees (x > 10) and all rocks
- **Bug found & fixed**: the refactored `place()` was only checking `tgt.userData.noShadowCast` and completely ignoring `o.noShadow` — every `noShadow: true` call on trees/rocks was silently doing nothing
- Fix: `var panelCastShadow = !o.noShadow && !(tgt && tgt.userData && tgt.userData.noShadowCast)`

#### 4. Removed Pomodoro Timer
- User no longer uses the 25-minute Pomodoro timer
- Deleted: 8 lines of CSS (`#pomodoroBox`, `#pomoDial`, `#pomoLabel` styles), the HTML `#pomodoroBox` div, and the entire 40-line JS IIFE

### Key Insights
- `material.visible = false` while `mesh.visible = true` in Three.js r128: the shadow map pass uses a depth material, NOT the regular material — so the invisible box still casts a shadow. Clean way to make shadow-only casters.
- When refactoring a function, be careful not to drop existing option checks — the `o.noShadow` path was lost silently when the `tgt.userData` approach was introduced.
- A 45° (Math.PI/4) rotation on a building gives the best 3/4 isometric look — both front (door) and one side wall are visible simultaneously.
