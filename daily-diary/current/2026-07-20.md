### Key Insights & Learning
- Runtime skills should have one canonical source; installer/catalog copies need deliberate synchronization to avoid drift.
- Rendered markup and syntax checks do not prove an interactive feature works. The exact user click path must be exercised in a real browser.
- Layered Phaser and Three.js interfaces require explicit attention to pointer propagation, event phase, raycasting, and fallback hit areas.
- A skill definition and its runtime backend are different things: Mulahazah's `SKILL.md` exists, but its persistent observer system is not installed.
