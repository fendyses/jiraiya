## 2026-05-22 - ican2u Tetapan Sistem Menu Visibility Fix (~16:03 GMT+8)

### Session Summary
- **Repo:** `ican2u` (`/Users/pairofspades/Sites/ican2u`)
- **Focus:** Diagnose and fix why "Tetapan Sistem" menu was not visible after being created
- **Outcome:** ✅ Root cause found and fixed — menu now visible

---

### Problem
User (logged in as Administrator role) could not see the **Tetapan Sistem → Tetapan Industri** menu under `/ceofaculty` even though menu items existed in the database.

### Investigation Path
1. Confirmed menu items existed in DB (IDs 160, 161) ✅
2. Confirmed `app_function` record existed (ID 125, path: `ceofaculty/tetapan-sistem/tetapan-industri`) ✅
3. Discovered the menu is rendered from **Meilisearch**, not directly from MySQL
   - `MenuRepository::getMenu()` uses a Meilisearch client to query the `menu_item` index
   - New DB records were **never synced** to Meilisearch after being inserted
4. Also discovered Administrator role (id: 1) was missing the 4 permissions for Tetapan Industri
   - Menu uses `addIfCan('.view')` — so missing permission = menu hidden

### Fixes Applied

| Fix | What |
|-----|------|
| `php artisan scout:import App\Models\MenuMgmt\MenuItem` | Synced all 161 menu items to Meilisearch — new entries now visible |
| DB insert into `tbl_role_has_permissions` | Added all 4 permissions (255–258) to Administrator role (id: 1) |

### Key Lesson Learned
**Whenever menu items are added to the DB, `scout:import` for `MenuItem` must be run** — otherwise Meilisearch serves the stale menu and new items never appear, regardless of DB state.
